# X12 Legacy Entities v0

## Зачем этот документ

В `X12` стало ясно,
что привычные сущности старого Linux graphics world
уже не являются базовыми сущностями новой модели.

Речь прежде всего про:

- `window manager`
- `compositor manager`

## Главный тезис

`window manager` и `compositor manager`
в `X12` должны читаться как `legacy entities`.

То есть не как первичные сущности мира,
а как старые имена для функций,
которые в `X12` уже разложены по слоям.

## Почему они становятся legacy

В старых системах эти сущности были большими комками власти.

Они одновременно держали:

- форму
- stacking
- видимость
- overlap
- часть routing truth
- часть presentation truth

В `X12` это больше не нужно собирать в один ком.

## Как они распадаются в X12

### `table`

Сюда уходит структурная часть старых менеджеров:

- relation truth
- легальность формы
- адресуемость
- claims и bindings
- первичная раскладка

### `crystall`

Сюда уходит видимая и формообразующая часть:

- shape of form
- stacking
- overlap
- visibility
- packet filtering по перекрытию
- composition-ready organization

Именно здесь packet'ы нижних форм могут отбрасываться,
если сверху их уже перекрыла другая форма.

### `manifest`

Сюда уходит только финальная реализация:

- present
- scanout
- final visual outcome

## Практический вывод

То, что раньше называлось:

- `window manager`
- `compositor manager`

в `X12` больше не должно строиться как отдельные царские сущности.

Правильнее читать это так:

- часть их функций живет в `table`
- часть живет в `crystall`
- часть завершается в `manifest`

## Особо важный момент

Отбрасывание packet'ов по перекрытию окон
должно происходить не в `table`,
а в `crystall`.

Почему:

- `table` знает легальность
- `crystall` знает реальную видимость формы

То есть:

- `table` может сказать, что форма допустима
- `crystall` решает, доживут ли ее packet'ы до `manifest`

## Короткая формула

`window manager` и `compositor manager` в `X12`
не базовые сущности,
а legacy-имена для функций,
которые теперь правильно разложены по слоям.
